Privacy & Security

MediConnect's enterprise-grade cybersecurity framework protecting your health data — DISHA, PDPB & ISO 27001 compliant.

DISHA Compliant ISO 27001 PDPB 2023 PCI-DSS IT Act 2000

1. Data Encryption

🔒 Data in Transit

TLS 1.3 with HSTS headers ensures all API calls and web traffic are encrypted end-to-end. Certificates renewed via Let's Encrypt.

💾 Data at Rest

AES-256-GCM encryption for database columns containing PII and medical records. SQLite WAL mode with encrypted volumes in production.

🔑 UPI & Payments

UPI IDs are tokenised — never stored as plain text. RSA-2048 key exchange for payment tokens. PCI-DSS Level 1 compliant processing.

📹 Telemedicine

WebRTC with DTLS-SRTP for video streams. Rooms are ephemeral — no video is stored. E2E encrypted via peer-to-peer channels.

Tools & Stack: OpenSSL · Python cryptography library · Fernet symmetric encryption · HashiCorp Vault for secrets management · AWS KMS (production)

2. Authentication System

🔐 Multi-Factor Auth (MFA)

TOTP-based MFA (Google Authenticator / Authy) for doctors and admins. OTP via SMS (Twilio) for patients. Backup recovery codes generated at registration.

🎫 Session Management

JWT tokens (RS256, 1h expiry) with refresh token rotation. Sessions invalidated on role change, password reset or suspicious login. Redis session store.

🛡️ Password Security

Bcrypt (cost factor 12) hashing. Password strength enforced (min 12 chars, mix). Credential stuffing protected via HaveIBeenPwned API check at registration.

🔑 OAuth 2.0 / ABHA

Sign in with Ayushman Bharat Health Account (ABHA) ID. OAuth 2.0 / OIDC via Google & ABDM. Federated identity with NHA guidelines.

Tools: Flask-Login · Flask-JWT-Extended · PyOTP · python-bcrypt · Redis · ABDM OAuth API

4. Database Security

🔒 SQL Injection Prevention

SQLAlchemy ORM with parameterised queries throughout. Zero raw SQL. Input validation via WTForms + Marshmallow schemas. WAF rules for SQLi patterns.

🗄️ Backup & Recovery

Automated encrypted backups every 6 hours (AES-256). 30-day retention with offsite replication. RTO: 4h, RPO: 6h for disaster recovery.

📊 Audit Logging

All CRUD operations on patient records are timestamped and logged with user identity, IP, and action. Logs stored in append-only, tamper-evident format.

🔐 Least Privilege

DB user accounts have minimal permissions per service. Separate read-only replica for analytics. No application uses root DB credentials. Secrets in Vault.

Tools: SQLAlchemy ORM · SQLCipher (SQLite encryption) · pgBadger / SQLite Audit · AWS RDS encrypted snapshots (production)

5. Indian Legal Compliance

📜 DISHA 2018

Digital Information Security in Healthcare Act. EHR stored with consent. Health data classified as sensitive information. Breach notification within 72 hours.

🔏 PDPB 2023

Personal Data Protection Bill. Explicit patient consent for data processing. Right to access, correct, and erase data. Data localisation in India-based servers.

💻 IT Act 2000

Section 43A (data protection), Section 72A (breach penalty). Reasonable security practices per CERT-In standards. Incident reporting to CERT-In within 6 hours.

🏥 NMC / IMC Act

Telemedicine Practice Guidelines (2020). Registered doctors only for prescriptions. Patient consent for telemedicine. Location-based prescribing restrictions.

💳 RBI / NPCI (UPI)

NPCI UPI framework compliance. No storage of UPI PINs or payment credentials. Transaction limits per RBI guidelines. 2FA mandatory for payments.

🏛️ ABDM / ABHA

Ayushman Bharat Digital Mission compliant. ABHA ID integration for patient identity. Health Records (HR) module follows FHIR R4 standard. Consent artefact based sharing.

6. Protection Against Cyberattacks

HIGH RISK SQL Injection

✅ ORM parameterised queries · ModSecurity WAF · Input validation · Prepared statements only

HIGH RISK XSS Attacks

✅ Jinja2 auto-escaping · Content-Security-Policy headers · DOMPurify on frontend · X-XSS-Protection header

MED RISK CSRF

✅ Flask-WTF CSRF tokens on all forms · SameSite=Strict cookies · Origin header validation

HIGH RISK DDoS / Brute Force

✅ Cloudflare DDoS protection · Flask-Limiter (rate limiting) · Account lockout after 5 fails · reCAPTCHA v3

MED RISK Data Breach

✅ AES-256 at rest · TLS 1.3 in transit · Tokenised PII · Honeypot decoy records · Real-time anomaly alerts via SIEM

HIGH RISK Ransomware

✅ Immutable backups (S3 Object Lock) · Network segmentation · Endpoint protection (CrowdStrike) · IR playbook

LOW-MED API Abuse

✅ API key authentication · Rate limiting per endpoint · JWT expiry · Swagger hidden in production · API versioning

MED RISK Insecure Uploads

✅ File type whitelist (PDF/JPEG/PNG only) · ClamAV virus scan on upload · Signed S3 URLs with 15-min expiry · No execution directory

Security Stack: Cloudflare · ModSecurity WAF · OWASP ZAP (pentesting) · Bandit (Python SAST) · Snyk (dependency scanning) · Nikto · Nmap · OWASP Top 10 checklist · ELK Stack (SIEM)
